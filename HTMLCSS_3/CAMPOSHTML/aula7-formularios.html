Aula 7: Formulários e Campos de Texto

Bem-vindos à nossa aula sobre formulários HTML, o principal meio de interação entre usuários e aplicações web. Aprenderemos como estruturar formulários eficientes e acessíveis para coletar informações dos usuários.

Por que usar formulários?

Formulários são a principal forma de interação entre o usuário e a aplicação web, permitindo a coleta de dados de maneira estruturada e organizada.

Exemplos de uso no dia a dia:

  • Cadastro de usuários: coleta de informações pessoais para criação de contas e perfis.
  • Login: autenticação de usuários para acesso a áreas restritas.
  • Comentários: espaço para feedback, avaliações e interações sociais.
  • Pedidos em lojas virtuais: preenchimento de dados de entrega e pagamento.
  • Sistemas de busca: campos de pesquisa em sites.

A Estrutura Básica do Formulário

A Tag <form>

A tag <form> é o contêiner principal que agrupa todos os campos do formulário. Ela é o elemento fundamental para qualquer interação baseada em entrada de dados — funciona como um "envelope" que contém todos os campos, seleções e botões.

Atributos essenciais da tag <form>:

  • action: define para qual URL os dados serão enviados após o usuário clicar em "Enviar".
  • method: especifica o método HTTP a ser usado. Os dois mais comuns são:
    • GET — os dados aparecem na URL (visíveis). Usado em buscas.
    • POST — os dados vão no corpo da requisição (mais seguro). Usado em cadastros e logins.

Exemplo de estrutura básica de um formulário:



<form action="/enviar-dados.php" method="post">

  <!-- Aqui vão os campos do formulário -->

  <!-- Botão de envio -->
  <input type="submit" value="Enviar">

</form>

O Campo de Entrada de Dados

A tag <input> é o elemento mais versátil dos formulários. Seu comportamento muda conforme o valor do atributo type. O atributo name é fundamental: ele define o identificador do campo para que o servidor possa reconhecer os dados enviados. Sem ele, o valor inserido pelo usuário não será processado corretamente.

Tipos de input mais comuns:

  1. type="text" — para textos curtos como nome, sobrenome, endereço.
    <input type="text" name="nome">
  2. type="email" — para endereços de e-mail. O navegador valida automaticamente o formato (ex: precisa conter "@").
    <input type="email" name="email">
  3. type="password" — para senhas. O texto digitado é ocultado com asteriscos ou pontos.
    <input type="password" name="senha">
  4. type="number" — para números inteiros ou decimais. Exibe controles de incremento e decremento.
    <input type="number" name="quantidade">
  5. type="date" — exibe um seletor de calendário para escolher uma data.
    <input type="date" name="data_nascimento">
  6. type="time" — permite selecionar um horário específico.
    <input type="time" name="horario">
  7. type="range" — cria um controle deslizante para selecionar valores dentro de um intervalo.
    <input type="range" name="volume" min="0" max="100">
  8. type="submit" — cria o botão de envio do formulário.
    <input type="submit" value="Enviar">

Atributos Adicionais Importantes

Além do type e do name, os campos de entrada aceitam outros atributos úteis:

  • placeholder — texto de exemplo exibido dentro do campo quando ele está vazio. Desaparece quando o usuário começa a digitar.
    <input type="text" placeholder="Seu nome aqui">
  • required — torna o campo obrigatório. O navegador impede o envio do formulário se o campo estiver vazio.
    <input type="email" required>
  • min e max — definem os valores mínimo e máximo aceitos (usados com type="number" ou type="range").
    <input type="number" min="1" max="100">

Identificando os Campos: a Tag <label>

A tag <label> associa uma descrição textual a um campo do formulário. Ela melhora a experiência do usuário e é essencial para acessibilidade.

Benefícios de usar <label>:

  • Acessibilidade: leitores de tela conseguem identificar o propósito de cada campo, auxiliando pessoas com deficiência visual.
  • Usabilidade: ao clicar no texto do rótulo, o campo correspondente recebe o foco automaticamente — muito útil em dispositivos móveis.
  • Clareza: torna o formulário mais organizado e compreensível.

A associação entre o <label> e o <input> é feita pelos atributos for (no label) e id (no input), que devem ter valores idênticos.

Exemplo:


<label for="nome">Nome:</label>
<input type="text" id="nome" name="nome">

<!--
  O atributo for="nome" no label aponta para o id="nome" no input.
  Isso cria a associação técnica entre rótulo e campo.
-->

Dica importante: mesmo que você aplique estilos que visualmente mostrem a relação entre rótulos e campos, a associação técnica via for e id é essencial para usuários de tecnologias assistivas.
